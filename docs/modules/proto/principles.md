# Proto 模块原理文档（Phase 1）

## 1. 契约优先
- `vote.proto` 是内部通信的唯一来源，服务端和客户端均基于该契约生成代码。
- 通过 `VoteRequest.topic_name` 与 `VoteResponse.results` 保持接口语义简单且稳定。

## 2. 包路径一致性
- `go_package` 采用模块绝对路径，避免跨目录引用时出现包路径歧义。
- 固定 `pb` 包名，保证业务层导入表达一致。

## 3. 可重复生成
- `scripts/gen_proto.sh` 固定了 proto 输入目录、输出目录与插件路径，确保不同机器生成结果一致。
- 插件路径显式声明可规避 `PATH` 配置不一致导致的构建失败。

## 4. 变更影响面
- 对消息字段变更会影响 gRPC 交互双方、业务逻辑解析及文档索引。
- 对 RPC 方法变更会影响服务注册、客户端调用和错误处理路径。
