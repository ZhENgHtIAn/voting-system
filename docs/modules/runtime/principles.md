# Runtime 模块原理文档（配置与 Redis）

## 1. 配置驱动
- 配置文件将运行参数与业务代码解耦，便于本地环境与后续部署环境切换。
- 默认值机制保证最小可运行，显式校验保证配置错误尽早暴露。

## 2. 固定话题约束
- 固定 3 话题来自业务决策，配置层负责强约束（数量与唯一性）。
- 该约束在启动期校验，防止运行期出现不可预期话题集。

## 3. Redis 连接与初始化
- Redis 客户端集中创建，避免业务层散落连接参数。
- 启动时执行 `Ping` 与话题字段初始化，降低首次请求冷启动风险。

## 4. 错误隔离
- 配置错误在加载阶段失败并阻止服务启动。
- Redis 基础能力错误在基础设施层集中返回，业务层只处理语义错误与调用结果。
