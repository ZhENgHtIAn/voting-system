# GrpcServer 模块原理文档（Phase 2）

## 1. 业务职责边界
- GrpcServer 仅承担投票核心逻辑与 Redis 交互，不包含前端和 HTTP 适配层逻辑。
- 通过 gRPC 契约实现与上游服务解耦。

## 2. 并发安全与一致性
- 投票写入使用 Redis `HINCRBY`，单命令原子完成并发计数更新。
- 返回结果通过 `HGETALL` 读取，确保调用方拿到全量最新视图。

## 3. 严格校验策略
- 采用固定 3 话题白名单，`CastVote` 对非法话题直接返回 `InvalidArgument`。
- 读取结果时仅暴露白名单话题，避免脏字段污染响应。

## 4. 启动时防御性初始化
- 服务启动即执行 Redis 连通性检查，快速失败。
- 预初始化固定话题到 Hash，保证首次查询结果稳定。

## 5. 错误语义分层
- 参数层错误：`codes.InvalidArgument`。
- 存储与内部执行错误：`codes.Internal`。
- 该分层便于上游网关映射为稳定的 HTTP 错误响应。
